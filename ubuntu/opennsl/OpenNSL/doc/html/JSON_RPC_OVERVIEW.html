JSON RPC Overview

Remote Procedure Call (RPC) is an inter-process communication that allows multiple processes/applications to interface with OpenNSL. As part of this, OpenNSL library is enhanced to support JSON-RPC, where server expects messages from the clients in JSON data format. Upon receiving the message, it decodes the message, invokes the corresponding driver API and returns the response/status back to the client application in JSON data format. The advantage of JSON-RPC is that it' a stateless, light-weight RPC protocol and allows client applications to be written in different programming languages.

OpenNSL RPC server is initialized as part of the OpenNSL driver initialization. This includes:

  • Reads the RPC configuration from RPC server configuration file named "rpc_server_config.cfg". It allows customers to choose a desired port for RPC server port.
  • Uses default server port of 7887, if the system is unable to fetch the RPC server port.
  • Opens a socket on the RPC server port.
  • Listens to requests from RPC clients.

OpenNSL JSON-RPC accepts requests in JSON data format only. This means that the client applications should be responsible for sending and parsing the JSON messages.

OpenNSL JSON-RPC currently supports JSON-APIs that are specific to Broadview Buffer Statistics (BST) application. The document OpenNSL Broadview JSON RPC API describes JSON API data format for Broadview application to interact with OpenNSL using JSON-RPC messaging, where Broadview application is running as a client application with server functionality running as part of the OpenNSL.
